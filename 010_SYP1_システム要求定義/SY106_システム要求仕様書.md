# 010_SYP1_システム要求定義

## SY106_システム要求仕様書

### 1. はじめに

#### 1.1 文書の目的
本システム要求仕様書は、「スマートグリーンハウスIoTシステム」のシステム全体にわたる要求事項を明確に定義することを目的とします。これにより、システム開発の全ての段階において、関係者間での共通理解を促進し、開発されるシステムがビジネス目的とユーザーニーズを確実に満たすことを保証します。

#### 1.2 対象システム
本仕様書が対象とするシステムは、中規模ビニールハウス向けのIoTベースの環境制御システムです。このシステムは、各種センサーデータをリアルタイムで収集し、アクチュエータを自動制御することで、最適な栽培環境を維持し、生産効率の向上と作業負担の軽減を目指します。将来的なAIによる環境制御最適化機能も本仕様書で展望します。

#### 1.3 適用範囲
本仕様書は、システムの機能要件、非機能要件、およびインターフェース要件を網羅します。ハードウェアの詳細設計、特定のアルゴリズム実装、運用手順、およびテストケースの詳細は、別途作成される設計書やテスト仕様書に委ねられます。

### 2. システム概要

#### 2.1 システムのビジョン
「人が付きっきりで管理する農業」から「データに任せて見守る農業」へ転換し、持続可能で高効率なスマート農業を実現します。

#### 2.2 システムの主要機能
*   **環境データ収集**: 温度、湿度、CO2、土壌水分、日射量センサーからのデータ収集。
*   **環境自動制御**: 換気窓、ミスト、灌水、暖房、照明のアクチュエータを自動で制御。
*   **データ蓄積と可視化**: 収集データをクラウドに蓄積し、ダッシュボードでリアルタイム表示と履歴表示。
*   **異常検知とアラート通知**: 設定された閾値に基づく異常検知と、スマートフォン/PCへの通知。
*   **遠隔監視と手動制御**: クラウド経由でのシステム状態監視とアクチュエータの手動操作。
*   **AIによる制御最適化**: 蓄積されたデータに基づき、最適な環境制御を提案・実行（将来機能）。

### 3. 機能要件

#### 3.1 環境データ収集要件
| ID | 機能要件 | 説明 | 優先度 |
|----|----------|------|--------|
| SYR-FUN-010 | 各種センサーデータ収集機能 | 温度、湿度、CO2、土壌水分、日射量センサーから、指定された間隔で環境データを収集できること。 | 高 |
| SYR-FUN-011 | データ送信機能 | 収集したセンサーデータを、LoRaWANまたはWi-Fi経由で通信ゲートウェイに送信できること。 | 高 |

### 4. 非機能要件

| ID | 非機能要件 | 説明 | 優先度 |

### 6. データ要件

#### 6.1 収集データ項目
| データ項目名 | 型 | 単位 | 備考 |
|--------------|----|------|------|
| 温度 | Float | ℃ | 測定範囲: -10℃ 〜 60℃ |
| 湿度 | Float | %RH | 測定範囲: 0% 〜 100% |
| CO2濃度 | Float | ppm | 測定範囲: 0ppm 〜 5000ppm |
| 土壌水分 | Float | % | 測定範囲: 0% 〜 100% |
| 日射量 | Float | W/m² | 測定範囲: 0 W/m² 〜 2000 W/m² |
| タイムスタンプ | DateTime | - | データ収集時刻 |
| デバイスID | String | - | データ収集元デバイスの識別子 |

#### 6.2 制御設定データ項目
| データ項目名 | 型 | 単位 | 備考 |
|--------------|----|------|------|
| 目標温度 | Float | ℃ | 自動制御の目標温度 |
| 目標湿度 | Float | %RH | 自動制御の目標湿度 |
| 目標CO2濃度 | Float | ppm | 自動制御の目標CO2濃度 |
| 目標土壌水分 | Float | % | 自動制御の目標土壌水分 |
| 照明スケジュール | JSON | - | 照明のオン/オフ時間、照度設定など |
| アクチュエータ状態 | Boolean | - | 手動制御時のアクチュエータON/OFF状態 |
